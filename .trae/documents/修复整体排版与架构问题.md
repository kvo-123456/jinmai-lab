## 问题分析

通过分析代码，我发现了导致页面排版问题的主要原因：

1. **重复标题**：在 `SidebarLayout` 的 header 中显示了页面标题，同时在 `Explore` 页面中又使用了 `GradientHero` 组件显示相同的标题
2. **重复搜索框**：在 `SidebarLayout` 的 header 中提供了搜索功能，同时在 `Explore` 页面中又添加了一个搜索框
3. **布局层次不清晰**：页面元素堆叠，缺乏明确的视觉层次

## 修复方案

### 1. 优化 `SidebarLayout` 组件
- 保留 header 中的标题显示，确保一致的导航体验
- 移除 header 中的搜索框，避免功能重复
- 调整 header 样式，使其更简洁

### 2. 重构 `Explore` 页面
- 保留 `GradientHero` 组件，但修改其样式，使其作为页面的视觉引导而非重复标题
- 保留页面内容区的搜索框，优化其样式和位置
- 调整页面元素的间距和层次，提升整体可读性

### 3. 统一设计语言
- 确保页面元素的颜色、字体、间距等保持一致
- 优化响应式设计，确保在不同设备上都有良好的显示效果
- 增强页面的视觉层次感，使用阴影、边框等元素区分不同区域

## 预期效果

1. 页面标题不再重复，形成清晰的视觉层次
2. 搜索功能集中在页面内容区，更符合用户使用习惯
3. 页面布局更加简洁、有序，提升用户体验
4. 统一的设计语言，增强品牌一致性

## 实施步骤

1. 修改 `SidebarLayout.tsx`，移除 header 中的搜索框
2. 调整 `Explore.tsx` 中的 `GradientHero` 组件样式
3. 优化 `Explore` 页面的搜索框样式和位置
4. 调整页面元素的间距和层次
5. 测试响应式设计，确保在不同设备上都有良好的显示效果
